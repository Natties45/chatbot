OLS Chatbot Architecture

Dify + n8n + Ollama · OpenLandscape Cloud Self Service

🔄 Data Flow

  • • User → Browser → Caddy → Dify Chatbot
  • • Knowledge YAML → GitHub → n8n sync → Dify KB
  • • Dify → Ollama → Embeddings → Vector Store
  • • n8n monitors → LINE Notify alerts

🗂️ Two Repositories

  • • chatbot/ — infra compose, scripts, prompts
  • • selfservice-repo — canonical knowledge YAML (read-only)
  • • Knowledge = derived store → rebuild anytime

🔐 Security Posture

  • • n8n admin = SSH tunnel :5678 only
  • • Caddy exposes only Dify Web (port 80/443)
  • • All secrets in .env (gitignored) / n8n store
  • • Customer bot gated behind red-team review
